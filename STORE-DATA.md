# Декларации данных в консолях магазинов

Шпаргалка для форм «безопасность данных» в кабинетах сторов. Эти формы — **не страницы сайта**, они
заполняются в консоли каждого магазина и должны **совпадать** с соответствующей политикой
(`privacy-*.html`).

> ⚠️ Это рабочая заготовка, а не юридическое заключение. Перед отправкой сверьте с актуальными
> требованиями каждого стора — категории и формулировки они периодически меняют. Пункты, где есть
> неоднозначность, помечены **(уточнить)**.

## Что приложение реально собирает

Только обезличенную телеметрию. Личный контент (запросы, открытые слова, история) — **нет**.

| Данные | Назначение | Привязка к личности |
|---|---|---|
| События использования (действия, без текста) | Аналитика | Нет — анонимный id сервиса |
| Технические данные (версия приложения/ОС, модель, язык) | Аналитика | Нет |
| Примерная страна (из IP) | Аналитика | Нет |
| Отчёты о сбоях (тип, стек, версия, устройство) | Стабильность | Нет |
| Атрибуция установки (App Store: AppsFlyer, без IDFA/ATT) | Маркетинг | Анонимная |
| Рекламный идентификатор (только RuStore: AppMetrica / GAID) | Аналитика | по политике Яндекса |

## Google Play — форма Data Safety

- **Does your app collect or share user data?** Yes (собирает; «share» с обработчиками под контрактом
  Google в большинстве случаев НЕ считается sharing — **уточнить** под свой случай).
- Категории «Collected»:
  - **App activity → App interactions** (события использования).
  - **App info and performance → Crash logs**, **Diagnostics**.
  - **Device or other IDs** (анонимный id аналитики). 
  - **Location → Approximate location** — **(уточнить)**: страна из IP. Если включаете, помечайте
    «не для трекинга».
- Для каждой категории: **Collected = Yes**, **Shared = No** (обработчики), **Processed ephemerally = No**,
  **Required = No** (но рантайм-тумблера opt-out нет — отразите это честно).
- **Security:** «Data is encrypted in transit» = Yes; «Users can request data deletion» — данные
  анонимны и не удаляются адресно → механизм = удалить приложение (локальные данные стираются) +
  контакт `notononoto@gmail.com`.
- НЕ отмечать: точную геолокацию, контакты, фото, сообщения, рекламный id.

## App Store — App Privacy (ярлыки конфиденциальности)

- **Data Used to Track You: None.** На iOS IDFA не собирается и ATT-промпт не показывается (политика —
  без всплывашек). AppsFlyer работает через AdServices + собственный анонимный id, без IDFA → не
  «tracking». Перед сабмитом убедиться, что в сборке AppsFlyer IDFA/ATT действительно выключены.
- **Data Linked to You:** None (аналитика обезличена).
- **Data Not Linked to You:**
  - **Usage Data → Product Interaction**;
  - **Diagnostics → Crash Data, Performance Data**;
  - **Identifiers → Device ID** (анонимный install-id SDK/AppsFlyer) — **(уточнить)** категорию.
- Плюс в бинаре: **Privacy Manifest** (`PrivacyInfo.xcprivacy`) с required-reason API и манифестами
  сторонних SDK (Amplitude/PostHog/Sentry/AppsFlyer тащат свои) — это техническое требование, не форма.

## RuStore

- Обязательное поле — **ссылка на политику конфиденциальности** (`privacy-rustore.html`); модерация
  открывает её **из РФ** → проверьте доступность (GitHub Pages = Fastly, из РФ может тупить; при
  необходимости — зеркало на Yandex Object Storage, канон оставить на домене).
- AppMetrica по умолчанию может собирать рекламный идентификатор (GAID) — это отражено в
  `privacy-rustore.html` (раздел «Сервисы»); при сборке его можно отключить, если он не нужен.
- 152-ФЗ: при отсутствии сбора персональных данных и не-резидентстве разработчика обязанности оператора
  не возникают; остаётся риск модерации — формулировки политики держать аккуратными.
